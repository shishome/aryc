About the Archive

The Art Archive was created to house all the art I've commissioned over the years.

It's pretty self explanatory, but there are over {{storage.arycListTotal}} files in this repository that require storage space, active upkeep, and most importantly... a filter for 18+ and unmentionables. This place really isn't meant to be a super flashy archive, but a programming challange to myself!

v1 of the Archive was just an Apache directory containing subdirectories, later migrating over to an NGINX based directory (what an upgrade!). v2 was a full django-based application which required me to upload everything to a web server host and specify media files for consumption. It was a great system, but really hard to keep up to date overall. However, it eventually collapsed when my upstream datacenter provider lost the disk containing my data, and I had no backups. v3 was similar to v2, but it pulled directly from a caching server hosted in my basement. This was better, but was far more convoluted.

v4: Aryc

As time went on, it just became too much for me to upload, link, and move everything to and from the clunky v3 system. So, for v4 I went back to directories. But not just that, I created Aryc, which stands for.. something clever I think.. Art RepositorY Commands, I genuinely forget. Aryc scans the directory tree in the server, which is catagorized in a very specific manner, to retrieve and store meta information about the system. From what year it was created to who made it to what character its for. This data is then stored in YAML files adjacent to the directory so the data can then be read by a viewing software. During this time I learned how to use Node and Angular, which is what a majority of this whole system is powered by. It's essentially a flat-file database that updates itself.

Why share all this?

TBH, development on this Aryc system took about 6 months of my life between starting a new job and other projects, so to see it completed makes me really proud. And, I hope it inspires other people to make neat software to manage their art too! Which is why a lot of this system is open source for anyone to peruse! You can find links to the repos down in the credits for the main code repository and credits for each individual piece!

About ArycViewer
ArycViewer is a NodeJS-based Angular application that fetches YAML records containing the metadata of each art piece, catagorizes, and displays the art for view in a gallery format. It was created with @angular/cli.
About Aryc
Aryc is a NodeJS-based Command-line application that compiles YAML records containing the metadata of each art piece in the gallery. It was created with OCLIF.
